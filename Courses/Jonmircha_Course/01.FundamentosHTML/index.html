Etiquetas semánnticas estructurales

El div: es un contenedor sin significado semántico.
El header: es la cabecera de un sitio web
El main: define el contenido principal de una página web, solo puede exisiter un main por documento
El footer: Define el pie de página de una página web o sección.
El nav: es una sección dedicada a los enlaces de navegación.
El article: es una sección del contenido de la página, como una publicación de blog o foro, que se puede reutilizar de forma aislada. Define un bloque de contenido autónomo que puede existir en cualquier contexto. Puede tener su propio encabezado, pie de página, secciones... Útil para una lista de publicaciones de blog.
El aside: representa el contenido que solo está indirectamente relacionado con el contenido principal, como una barra lateral.
El section: Una sección que contiene elementos agrupados por tema, por ejemplo, un capítulo o un cuadro de pestañas.
El address: representa una información de contacto.

Etiquetas de Bloque VS Etiqutas de linea

Etiqueta de bloque por exelencia es el div, además define un bloque genérico de contenido, que no tiene ningún valor semántico. Se utiliza para elementos de diseño como contenedores.
Una etiqueta de bloque es aquella que genera saltos de linea con sus etiquetas hermanas.
Etiqueta de linea por excelentia es el span, además define un contenedor genérico de contenido en línea, que no tiene ningún valor semántico.
Esta etiqueta solo ocuapa el espacio necesario que tiene su contenido. Hola, soy otra span
